# 부재리스트 자동 작성 AI 구축 계획서

## 1. 프로젝트 개요

건축구조 설계 과정에서 구조도면(DWG)을 기반으로 **부재리스트를 자동으로 작성하는 AI 시스템**을 구축한다. 기존에는 구조 엔지니어가 수동으로 도면을 읽고 부재 정보를 정리했으나, AI를 통해 이 과정을 자동화하여 효율성과 정확성을 높이는 것이 목표이다.

---

## 2. 현재 워크플로우 분석 (AS-IS)

프레젠테이션 자료 기반으로 현재 건축구조 설계 과정은 다음과 같다:

### 2.1 건축설계 과정
1. **건축도면 작성** → 건축사 사무소에서 전기, 설비, 소방, 구조 등의 정보 포함
2. **도면털기 작업** → 구조기술사 사무소에서 구조설계에 필요한 정보(기둥, 벽체, 슬래브 등)만 남기고 나머지 제거
3. **구조도면 작성** → 털어진 도면에 보, 슬래브 등 프레임을 배치
4. **모델링** → Midas Gen 등 상용프로그램에서 xyz 좌표계로 구조물 모델링
5. **부재설계** → Design+, BEST, Excel 등으로 단일 부재(보, 슬래브 등) 설계
6. **부재리스트 작성** → 설계된 부재 정보를 각 구조사무소 양식에 맞춰 정리

### 2.2 핵심 과제
- DWG 파일은 벡터 형식으로 일반 파이썬 등에서 직접 인식이 어려움
- DWG → DXF 변환 시 수치좌표 형식으로 접근 가능
- 도면의 각 점, 선 등의 객체는 **Layer**로 그룹화되어 있음
- 각 건축사마다 Layer 명명 규칙이 다름

## 3. 목표 시스템 설계 (TO-BE)

### 3.1 전체 파이프라인

```
[구조도면.dwg] → [DWG 파싱] → [Layer 분류] → [구조요소 인식] → [부재 정보 추출] → [부재리스트 자동 생성]
```

### 3.2 단계별 구현 계획

#### Phase 1: DWG 파일 파싱 기반 구축 
- DWG/DXF 파일 읽기 환경 구축 (Python + ezdxf)
- ODA File Converter를 통한 DWG → DXF 변환 파이프라인
- Layer 구조 파악 및 매핑
- 기본 도형(선, 원, 텍스트 등) 엔티티 추출

#### Phase 2: 구조요소 인식 AI
- Layer 이름 기반 구조요소 분류 (기둥, 보, 슬래브, 벽체 등)
- 도면 내 텍스트(치수, 부재명, 규격) OCR/추출
- 좌표 기반 부재 위치 및 크기 추정
- 각 구조사무소의 Layer 명명 규칙 학습 (규칙 기반 + ML)

#### Phase 3: 부재 정보 매핑
- 추출된 구조요소와 부재설계 결과(Design+, BEST 출력) 매핑
- MGT 파일(.mgt)에서 부재 재질, 하중, 좌표 등 보조 정보 활용
- 부재 단면 크기, 배근 정보 등 추출

#### Phase 4: 부재리스트 자동 생성
- 각 구조사무소 양식 템플릿 관리
- 추출 정보 기반 부재리스트 자동 채움
- Excel/DWG 형태로 부재리스트 출력
- 검증 및 수정 인터페이스 제공

### 3.3 기술 스택

| 구분 | 기술 |
|------|------|
| 언어 | Python 3.10+ |
| DWG 파싱 | ezdxf (DXF), ODA File Converter (DWG→DXF) |
| AI/ML | PyTorch, scikit-learn, OpenAI API |
| 데이터 처리 | pandas, numpy |
| 시각화 | matplotlib, plotly |
| 출력 | openpyxl (Excel), python-pptx (참고자료) |

## 4. 리스크 및 고려사항

- **DWG 포맷 호환성**: DWG 버전에 따라 파싱 라이브러리 호환 문제 가능
- **Layer 표준화 부재**: 각 사무소마다 다른 명명 규칙 → 규칙 학습 필요
- **복잡한 도면 구조**: 블록 참조, 외부 참조(XREF) 등 복잡한 구조 처리
- **정확도**: 자동 추출 정보의 정확도 검증 메커니즘 필요
